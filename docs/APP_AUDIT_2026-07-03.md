## 2. Shop-owner side (dashboard)

**State:** solid after today's fixes. Dashboard, Products, Orders, Stock,
Categories, Keywords, Promotions, Billing, Settings, Studio all render and work.

Improvements, ordered by value:

- 🟠 **Currency coherence.** Stat cards and product cards mix `$` and `ALL`
  (e.g. "Total Revenue $0.00" beside "ALL 3,500"). Everything should display in
  the shop currency by default with one formatter; the exchange-rate conversion
  is for the storefront customer, not the owner.
- 🟠 **Onboarding wizard (Phase 9).** New accounts land on an empty dashboard
  with no guidance. React Joyride tour + a "getting started" checklist card
  (connect Instagram → sync → review products → open Studio → share link).
  This is the single biggest activation lever.
- 🟠 **Manual product creation flow** exists but is buried behind "Add Product";
  Phase 10 (AI caption + auto-post to Instagram + post↔product registry) turns
  it into a differentiator. Needs Meta App Review for `instagram_content_publish`.
- 🟠 **Orders: fulfillment pipeline UX.** Status changes are a dropdown; a
  kanban-ish board or one-click "advance status" + customer-notification email
  hook would match how sellers actually work through a day's orders.
- 🟢 **Global search** (header) currently searches little; wire it to products +
  orders + customers with grouped results.
- 🟢 **Empty states** are good on Products; give Orders/Promotions/Keywords the
  same treatment (explain + primary action).
- 🟢 **Notifications sidebar**: add read/unread state persistence and deep links
  to the relevant order/product.
- 🟢 **Image pipeline**: product images are hot-linked Instagram CDN URLs that
  expire; "Fix Images" exists, but a background re-host into `shop-assets`
  storage (edge cron) would remove the class of bug entirely.

## 3. Storefront side (customer)

**State:** rebuilt this week — modular blocks with 3–8 layout variants each,
composable product-detail page, sidebar-nav chrome, 4 footer variants, zero
horizontal overflow across 360→3440px, 8 distinct templates.

Improvements:

- 🟠 **Customer accounts + reviews (Phase 3.0)** — biggest missing storefront
  feature: shopper register/login, order history, verified reviews with owner
  replies. The `reviews` detail-block slot already exists in the page config.
- 🟠 **Search & filtering depth.** Products page has category/price filters;
  add text search (ILIKE on name/caption), sort options surfaced on mobile, and
  filter persistence in the URL (shareable filtered links).
- 🟠 **Checkout trust signals.** Add payment-method logos, delivery-time
  estimate, and a post-order confirmation page with order-tracking link —
  conversion basics for the Albanian market (COD + card via RaiAccept).
- 🟠 **Performance.** Product images render full-size; generate/request resized
  variants (Supabase storage transforms or an image CDN param) for cards vs.
  detail. Also lazy-load below-fold sections (blocks already reveal on scroll).
- 🟢 **SEO per shop**: per-product `<meta>`/OG tags + JSON-LD Product schema on
  the detail page; sitemap edge function per shop slug.
- 🟢 **Wishlist / recently-viewed rail** (RecentlyViewedContext already exists —
  surface it as a home block variant).
- 🟢 **Cart upsells**: "you may also like" inside the cart drawer using the
  existing related-products logic.
